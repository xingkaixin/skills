# 三重心智模型：深读

对一篇文章/播客实录/演讲/论文，或一个复杂决策，不做概括式的平铺总结，而是同时用三个视角深入加工：既当正方，又当反方，还要导出对行动的启发。

核心不是"总结这篇讲了什么"，而是"这篇的观点站得住吗、能延伸到哪、对我意味着什么"。

## 适用范围

- 硅谷技术播客 / 圆桌论坛实录
- 长文（技术、思想、行业分析）
- 带框架和方法论性质的演讲、论文
- 审视一个复杂决策或方案

不适用：查事实、短文本、用户只想要一句话摘要。

## 三个视角（缺一不可）

同一份材料，依次戴上三副眼镜。不是三段互不相干的评论，而是对同一批观点的三重加工。

### 1. 挑战者（Challenger / 批判性思想家）
找弱点。以最严格的反方立场审视：
- 核心论点的前提是否成立？有没有未说明的假设？
- 证据是否充分？是逸事、个例，还是系统证据？样本有没有偏差？
- 逻辑有没有跳步、以偏概全、把相关当因果？
- 边界条件是什么？在什么情况下这个观点会失效？
- 作者的立场/利益是否影响了结论（卖课、卖产品、立人设）？
- 有没有更简单的解释被忽略了？

### 2. 建构者（Builder / 框架构建者）
扩展观点。假设核心洞见成立，往前推：
- 把零散的观点抽象成一个框架/模型。
- 这个想法能推广到哪些原文没提的场景？
- 和哪些已有的概念、理论、其他文章能连接、印证或互补？
- 顺着作者的逻辑再往前走一步，会得到什么新结论？
- 缺失的拼图是什么？补上哪一块，整个论述会更完整？

### 3. 实践者（Practitioner）
导出行动。落到"对我/对读者有什么用"：
- 从中能提炼出哪些可操作的做法、清单、原则？
- 我现在的做法和它有什么出入？该改什么？
- 最小可执行的第一步是什么？
- 有哪些坑、前置条件、代价需要注意？
- 什么情况下我不该照搬？

## 执行流程

1. **通读全文**，先抓住作者的核心主张和论证骨架（而不是逐段复述）。
2. **一句话锚定** 原文的核心命题（用来贯穿三个视角）。
3. **依次跑三个视角**：挑战者 → 建构者 → 实践者。每个视角独立成段，观点鲜明，不和稀泥。
4. **交叉**：挑战者挑出的弱点，建构者可以尝试补强或绕开；实践者要考虑弱点带来的实操风险。
5. **收束**：给出你自己（作为综合三视角的独立判断者）对这篇的净评价——值不值得信、值得信到什么程度、最该记住的一点。

## 输出结构

```
## 核心命题
（一句话概括原文真正在主张什么）

## 挑战者视角
（找弱点：前提、证据、逻辑、边界、立场偏差。观点鲜明，逐条列。）

## 建构者视角
（扩展：抽象成框架、连接已有概念、往前推一步、补缺失拼图。）

## 实践者视角
（行动：可操作做法/清单、与现状的差距、最小第一步、注意的坑。）

## 净评价
（综合三视角的独立判断：可信度、最该记住的一点、什么情况下不适用。）
```

## 原则

- **不平铺、不复述**。如果某一段只是把原文换个说法重说一遍，删掉。三个视角都要产出原文里没有的东西。
- **观点要鲜明**。挑战者就狠狠挑，建构者就大胆推，实践者就落到具体动作。不要每句都"一方面……另一方面……"和稀泥。
- **保持诚实**。挑战者不为挑而挑，建构者不脑补作者没有的意思，实践者不给空泛口号。
- **独立思考**。三视角跑完，最后要有你自己的判断，而不是把三段拼在一起就完事。

## 可选：直接对话式 prompt（迁移用）

当用户想用这套框架分析他自己贴进来的材料时，可用如下开场，然后让用户粘贴全文：

> 我们将一起深入分析一份材料。请你不做概括式总结，而是同时以三重心智模型来加工：**挑战者**（找弱点）、**建构者**（扩展并抽象成框架）、**实践者**（导出可操作的启发）。请动态调整视角、层次丰富地展开，我们的讨论应当平等、深入、自主。准备好后，我把材料全文贴给你。

用途可从"分析文档"迁移到"讨论复杂问题""学习一个概念""审视一个决策"——把三视角相应改为对应问题的正方、反方、行动方即可。
